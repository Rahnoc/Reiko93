# 需要安裝的東西 For MacOS (Tahoe / apple silicon) (針對 CLI 開發)

## 1. VS Code

`VSCode-darwin-arm64.dmg` (apple silicon，不然請抓 universal 版)

### 1.1 Extensions

`Haxe`

或是

`Haxe Extension Pack (有 Haxe 與debugger 等東西的那包)`

<br>

## 2. Haxe 4.3.7 (2025-0509)

`haxe-4.3.7-osx-installer.pkg`

```bash
# (安裝結果確認用)
$ haxe --version
```

裝完主要有兩個東西，haxe (編譯/轉譯)，以及 haxelib (套件管理器)。


### 2.1 安裝 haxelib 的 Hashlink

```bash
$ haxelib install hashlink
```

目標為轉譯 .c 時， openFL/Lime 呼叫特定 HashLink 底層 API 時會用到，提供 Haxe 端的類別庫（Bindings）。

```bash
# (安裝結果確認用)
$ haxelib list
```

### 2.2 安裝編譯 C++用的 haxelib 的 hxcpp

```bash
$ haxelib install hxcpp
```


### 2.3 安裝 haxelib 的 Node.js (編譯時借lib)

```bash
$ haxelib install hxnodejs
```

如果 不是要在瀏覽器看JS，而是在node.js看，且有用到 Sys.println(); 你會需要裝這個。

然後 shell 執行時，要加個 lib 參數 hxnodejs

```bash
$ haxe -js main.js -main Main -lib hxnodejs
```

如果是寫在 *.hxml 裡面，因為要分成**多行**所以要寫成

```hxml
-main aa.bb.Main
-js bin/main.js
-lib hxnodejs
```

### 3.1 安裝 Homebrew 的 Hashlink
    
```bash
$ brew install hashlink
```
    
clang 編譯 .c 檔時，連結時會用到。 

連結libhl.dylib 這個動態連結庫，有 HashLink 的機制（如垃圾回收機制、基本的資料結構、系統 API 串接）

```bash
# (安裝結果確認用)
$ brew info hashlink
```

### 3.2 安裝 Node.js

```bash
$ brew install node
```

執行用。為了在 CLI 環境下透過 Makefile 或直接呼叫 node 來執行產出的 JS 檔案。

不然你要寫 html 去包，然後用瀏覽器開發者模式看。

# 5. 圖形化應用程式開發 (補充與未來發展)

(目前與本專案無關)

對於 UI ， vs code 可安裝 extension: lime

haxelib 安裝 openFL 與 lime。

之後建立專案時可以用 openFL 生成專案。 (會改採 project.xml 管理專案)

然後用 lime 執行。

```bash
$ haxelib run openfl create 專案名稱
$ haxelib run lime test html5
$ haxelib run lime test mac
```

### lime 相關三種執行法

```bash
$ lime test hl
```

(apple silicon) haxe 轉*.hl 後，跑在 hashlink 上面。

```bash
$ lime test hlc
```

haxe 轉譯 c （*.c)後，用clang 編譯，hashlink 提供動態函式庫等連結資源。 然後執行。

```bash
$ lime test mac
```

haxe 轉譯 CPP (*.cpp）後， hxcpp 叫 clang 編譯，hxcpp 提供動態函式庫等連結資源。 然後執行。

-----

lime test hlc 較快，適合快速原形與測試。 (如果不看hl)

lime test mac 較慢，但適合發布。 (C++)
