## 2. Haxe 4.3.7 (2025-0509)

> haxe-4.3.7-osx-installer.pkg

> bash (安裝確認)
>
> haxe --version

裝完主要有兩個東西，haxe (編譯/轉譯)，以及 haxelib (套件管理器)。


### 2.1 安裝 haxelib 的 Hashlink

> ```
> haxelib install hashlink
> ```

轉譯 .c 會用到 (hl/c)

> bash (安裝確認)
>
> haxelib list


### 2.2 安裝轉譯/編譯 C++用的 haxelib 的 hxcpp

> ```
> haxelib install hxcpp
> ```


### 2.3 安裝 haxelib 的 Node.js (編譯時借lib)

> ```
> haxelib install hxnodejs
> ```

如果有用到 Sys.println(); 你會需要裝這個。

然後 shell 執行時，要加個 lib 參數 hxnodejs

> haxe -js main.js -main Main -lib hxnodejs

如果是 *.hxml，因為要分成**多行**所以要寫成

```
-main aa.bb.Main
-js bin/main.js
-lib hxnodejs
```

### 3.1 安裝 Homebrew 的 Hashlink
    
> ```
> brew install hashlink
> ```
    
clang 編譯 .c 檔時，連結時會用到。 

連結libhl.dylib 這個動態連結庫，有 HashLink 所有的核心靈魂（如垃圾回收機制、基本的資料結構、系統 API 串接）

> bash (安裝確認)
>
> brew info hashlink

### 3.2 安裝 Node.js

> bash (安裝確認)
>
> brew install node

執行用。為了執行時 makefile 讓東西在上面跑。

不然你要寫 html 去包，然後用瀏覽器開發者模式看。

# 5. 圖形化應用程式開發

(目前與本專案無關)

對於 UI ， vs code 可安裝 extension: lime

haxelib 安裝 openFL 與 lime。

之後建立專案時可以用 openFL 生成專案。

然後用 lime 執行。

```
haxelib run openfl create 專案名稱

haxelib run lime test html5

haxelib run lime test mac
```

### lime 相關三種執行法

> lime test hl

(apple silicon) hashlink 轉*.hl 後 偷偷用rosetta 2跑


> lime test hlc

hashlink 轉 c （*.c)後，用clang 編譯。


> lime test mac

與 hashlink 無關。 hxcpp 轉 CPP (*.cpp），用 clang 編譯。

-----

lime test hlc 較快，適合快速原形與測試。

lime test mac 較慢，但適合發布。 (C++)
